# 中文简历技能

使用本技能时，执行标准化流程并输出可复核产物。不要假设岗位类型，始终以用户提供的目标JD为准。

## 输入

- 基础简历 Markdown（UTF-8，推荐）
- 目标岗位 JD 文本（可来自招聘网站、企业官网、用户粘贴）
- 目标岗位名称（命令行参数 `--target-role`）

如用户只有 Word 简历，先执行：

```bash
bash skills/中文简历技能/脚本/Word转Markdown.sh 简历.docx 简历.md
```

## 执行流程

1. 读取基础简历与JD文本。
2. 提炼岗位关键词与核心职责（调研层）。
3. 进行经历匹配与改写（写作层）：
   - 使用 STAR 思路组织内容。
   - 每段经历输出 2-3 条 bullet。
   - 每条 bullet 使用固定句式：`\textbf{四字标签}：过程；关键指标：\textbf{量化}`。
4. 生成 LaTeX 与 HTML（交付层）。
5. 编译 PDF。
6. 输出调研与校准报告，供人工确认。

## 输出

- `优化后简历.json`
- `中文简历.tex`
- `中文简历.pdf`
- `中文简历网页.html`
- `岗位调研与简历校准报告.md`

## 质量门槛

- 结构完整：每段经历有 2-3 条 bullet。
- 结果导向：每条 bullet 包含动作与结果信息。
- 量化优先：优先写入可解释的数字证据（比例、金额、数量、时长）。
- 真实性可复述：报告中列出人工确认项，投递前必须复核。

## 命令示例

```bash
python3 skills/中文简历技能/脚本/生成简历流程.py \
  --base-resume skills/中文简历技能/素材/样例/雷军_基础简历样例.md \
  --jd-file skills/中文简历技能/素材/样例/岗位JD样例.txt \
  --out-dir skills/中文简历技能/输出/最终版 \
  --target-role "金融管培生" \
  --open-html
```

## 样例说明

- `素材/样例` 仅用于演示流程，不代表岗位限制。
- 观众应替换为自己的基础简历与目标JD后再生成正式版本。
